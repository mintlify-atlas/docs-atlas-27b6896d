The Hermes Labyrinth plugin exposes a read-only FastAPI router mounted at `/api/plugins/hermes-labyrinth/` by the Hermes dashboard plugin system. Every endpoint uses `GET` — the API never starts, stops, mutates, or creates Hermes sessions. Because the plugin runs locally on the same machine as the Hermes dashboard, no authentication is required.

**Base URL**

```
http://127.0.0.1:9119/api/plugins/hermes-labyrinth
```

## Endpoints

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/health` | Plugin health check and data source availability |
| `GET` | `/journeys` | List recent journeys with pagination |
| `GET` | `/journeys/{journey_id}` | Journey detail with guideposts and crossing count |
| `GET` | `/journeys/{journey_id}/crossings` | All crossings and threads for a journey |
| `GET` | `/skills` | Skill inventory across all source directories |
| `GET` | `/cron` | Scheduled job inventory with state and last run |
| `GET` | `/guideposts` | Recent automated observations across journeys |
| `GET` | `/reports/{journey_id}.json` | Full journey export as JSON |
| `GET` | `/reports/{journey_id}.md` | Full journey export as Markdown |

<Note>
  All responses include a `generated_at` field containing a Unix timestamp (float) recording when the response was assembled.
</Note>
