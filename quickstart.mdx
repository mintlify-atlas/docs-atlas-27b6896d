Hermes Labyrinth installs directly into the Hermes user plugin directory. No build step and no changes to the Hermes dashboard source are required — clone the repository, start or restart the dashboard, and the **Labyrinth** tab appears automatically.

<Note>
  Hermes Labyrinth is read-only. It never starts, stops, resumes, mutates, or creates Hermes sessions.
</Note>

## Install the plugin

<Steps>
  <Step title="Clone into the Hermes plugin directory">
    Create the plugin directory if it does not exist, then clone the repository into it:

    ```bash
    mkdir -p ~/.hermes/plugins
    git clone https://github.com/stainlu/hermes-labyrinth.git ~/.hermes/plugins/hermes-labyrinth
    ```
  </Step>

  <Step title="Start or restart the dashboard">
    If the Hermes dashboard is not running, start it:

    ```bash
    hermes dashboard
    ```

    The dashboard discovers plugins at startup and registers the **Labyrinth** tab automatically.
  </Step>

  <Step title="Rescan plugins if the dashboard is already running">
    If the dashboard was already running before you cloned the plugin, trigger a rescan without restarting:

    ```bash
    curl http://127.0.0.1:9119/api/dashboard/plugins/rescan
    ```
  </Step>

  <Step title="Open the Labyrinth tab">
    In the Hermes dashboard, select the **Labyrinth** tab. You should see the Journey Index listing recent sessions, cron runs, and delegated work.
  </Step>
</Steps>

## Optional: apply the theme scaffold

Hermes Labyrinth ships an optional dashboard theme file. To install it:

```bash
mkdir -p ~/.hermes/dashboard-themes
cp ~/.hermes/plugins/hermes-labyrinth/theme/hermes-labyrinth.yaml ~/.hermes/dashboard-themes/
```

Restart the dashboard after copying the theme file for it to take effect. The theme is cosmetic — removing it does not affect plugin behavior.

## Notes on fresh installs

<Tip>
  On a fresh Hermes install, the Journey Index may show sparse or empty data until sessions, skills, or cron jobs exist. Run a few Hermes sessions and reload the Labyrinth tab to populate the views.
</Tip>

Crossings are inferred from existing Hermes session messages. Some durations, model switches, tool timings, and subagent links depend on what Hermes has recorded in local state. Missing data is shown explicitly rather than fabricated.
