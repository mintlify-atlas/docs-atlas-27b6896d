Hermes Labyrinth is designed to work with no configuration. Clone the repository into the Hermes plugin directory, start the dashboard, and the **Labyrinth** tab appears automatically. This page documents the two optional configuration surfaces: the plugin manifest (read-only, shipped with the plugin) and the optional theme scaffold you can copy to customize the surrounding Hermes dashboard appearance.

## Plugin manifest

The plugin manifest at `dashboard/manifest.json` registers Hermes Labyrinth with the Hermes dashboard. You do not edit this file during a normal install — it is read by the Hermes plugin loader at startup.

```json dashboard/manifest.json
{
  "name": "hermes-labyrinth",
  "label": "Labyrinth",
  "description": "Journey, crossing, skill, cron, and guidepost observability for Hermes Agent.",
  "icon": "Activity",
  "version": "0.1.0",
  "tab": {
    "path": "/labyrinth",
    "position": "after:sessions"
  },
  "slots": ["header-right"],
  "entry": "dist/index.js",
  "css": "dist/labyrinth.css",
  "api": "plugin_api.py"
}
```

<AccordionGroup>
  <Accordion title="name">
    The plugin's internal identifier. Used by the Hermes plugin loader to uniquely address this plugin and to scope its API routes under `/api/plugins/hermes-labyrinth/`.
  </Accordion>
  <Accordion title="label">
    The display name shown in the Hermes dashboard tab bar. Set to `"Labyrinth"`.
  </Accordion>
  <Accordion title="icon">
    The icon rendered in the tab. Uses the Hermes dashboard icon set. Set to `"Activity"`.
  </Accordion>
  <Accordion title="version">
    The plugin version. Currently `"0.1.0"`.
  </Accordion>
  <Accordion title="tab.path">
    The client-side route the Labyrinth tab occupies. Set to `"/labyrinth"`.
  </Accordion>
  <Accordion title="tab.position">
    Where the tab appears in the dashboard navigation. `"after:sessions"` places the Labyrinth tab immediately after the Sessions tab.
  </Accordion>
  <Accordion title="slots">
    Dashboard layout slots the plugin occupies in addition to its tab. Currently `["header-right"]`, which reserves a header toolbar area for the plugin's export and search controls.
  </Accordion>
  <Accordion title="entry">
    The JavaScript bundle entry point, relative to the plugin directory. Resolved to `dashboard/dist/index.js` at runtime.
  </Accordion>
  <Accordion title="css">
    The stylesheet entry point, relative to the plugin directory. Resolved to `dashboard/dist/labyrinth.css` at runtime.
  </Accordion>
  <Accordion title="api">
    The Python backend module the Hermes plugin loader mounts under the plugin's API prefix. `plugin_api.py` registers all read-only routes.
  </Accordion>
</AccordionGroup>

## Optional theme scaffold

Hermes Labyrinth ships an optional dashboard theme file at `theme/hermes-labyrinth.yaml`. The theme applies an editorial ink-and-paper style to the surrounding Hermes dashboard — compact density, a dark paper background, graphite thread accents, and a restrained gold threshold color. It is purely cosmetic and does not affect plugin behavior.

To install the theme:

```bash
mkdir -p ~/.hermes/dashboard-themes
cp ~/.hermes/plugins/hermes-labyrinth/theme/hermes-labyrinth.yaml ~/.hermes/dashboard-themes/
```

Restart the Hermes dashboard after copying the file for the theme to take effect.

### Theme file reference

```yaml theme/hermes-labyrinth.yaml
name: hermes-labyrinth
label: Hermes Labyrinth
description: Editorial ink-and-paper dashboard theme for the Hermes Labyrinth extension.

layoutVariant: standard

palette:
  background: "#0e0d0a"
  midground: "#ece7d6"
  foreground:
    hex: "#c8d4e8"
    alpha: 0.08
  warmGlow: "rgba(201, 168, 87, 0.16)"
  noiseOpacity: 0.7

layout:
  radius: "0.25rem"
  density: compact

typography:
  fontSans: "Inter, ui-sans-serif, system-ui, sans-serif"
  fontMono: "JetBrains Mono, ui-monospace, SFMono-Regular, Menlo, Monaco, Consolas, monospace"
  fontDisplay: "Newsreader, Georgia, serif"
  baseSize: "14px"
  lineHeight: "1.5"
  letterSpacing: "0"
```

**Palette fields**

| Field | Value | Description |
|---|---|---|
| `background` | `#0e0d0a` | Near-black base surface for the dashboard shell. |
| `midground` | `#ece7d6` | Warm off-white used for card and panel surfaces. |
| `foreground.hex` | `#c8d4e8` | Cool blue-grey used for foreground elements. |
| `foreground.alpha` | `0.08` | Opacity applied to the foreground color in layered contexts. |
| `warmGlow` | `rgba(201, 168, 87, 0.16)` | Muted gold glow used for threshold and highlight accents. |
| `noiseOpacity` | `0.7` | Intensity of the paper-grain noise texture overlay. |

**Layout fields**

| Field | Value | Description |
|---|---|---|
| `radius` | `0.25rem` | Corner radius applied to cards, panels, and inputs. |
| `density` | `compact` | Reduces spacing throughout the dashboard shell. |

**Typography fields**

| Field | Value | Description |
|---|---|---|
| `fontSans` | Inter, system fallbacks | UI sans-serif stack for labels, metadata, and body copy. |
| `fontMono` | JetBrains Mono, system fallbacks | Monospace stack for code, identifiers, and tool names. |
| `fontDisplay` | Newsreader, Georgia, serif | Serif stack used for headings and journey titles. |
| `baseSize` | `14px` | Root font size for the dashboard shell. |
| `lineHeight` | `1.5` | Default line height. |
| `letterSpacing` | `0` | No additional letter spacing applied. |

<Note>
  Configuration is minimal by design. The plugin is read-only and self-contained — it reads local Hermes state but never writes to it, and the `dashboard/dist/` bundle is checked in so no build step is required after cloning.
</Note>
