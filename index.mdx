Hermes Labyrinth turns every Hermes Agent session into a navigable map. Prompts, tool calls, tool results, model switches, subagent delegations, approvals, memory events, context compressions, and scheduled runs all become crossings on a journey you can inspect, filter, and export — without ever mutating agent state.

## What Hermes Labyrinth shows you

Hermes Agent crosses many boundaries during autonomous work: user intent into tool action, local machine into network services, single agent into delegated subagents, short-term context into memory, and manual work into scheduled cron jobs. Labyrinth makes every one of those crossings visible.

## Read-only by design

<Note>
  Hermes Labyrinth never starts, stops, resumes, mutates, or creates Hermes sessions. It is a pure observer — safe to run alongside any active agent work.
</Note>

Reports, previews, and all API responses have secret redaction applied before display. Unknown fields are shown as unknown rather than fabricated.

<Steps>
  <Step title="Install the plugin">
    Clone into `~/.hermes/plugins/` and start or restart the Hermes dashboard.
  </Step>
  <Step title="Open the Labyrinth tab">
    Select the **Labyrinth** tab in the Hermes dashboard to see your journey index.
  </Step>
  <Step title="Explore a journey">
    Click any journey to open the Labyrinth Map and inspect individual crossings.
  </Step>
  <Step title="Export a report">
    Use the Reports view to download a redacted Markdown or JSON report for any journey.
  </Step>
</Steps>
